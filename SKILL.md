# 规则

- 每次会话只能有一个用户身份，uid 储存在会话变量 `sbti_uid` 中
- **答题顺序由 LLM 在内存中随机打乱**，每批 5 题，不调用任何脚本
- 测试未提交前不得输出结果图片或解读
- 历史记录以 `sbti_uid` 为键，存于 `<skill-base>/data/`
- 解读话术风格必须从 `<skill-base>/references/author-voice.md` 中提取，遵守该文件的语调约束
- 结果出图统一使用 `<skill-base>/scripts/generate_result_card.py` 生成 ljg-card 风格信息图，不直接展示原始图片

# 安装自检 ✋

**skill 安装时（首次初始化）自动执行此步骤，不需要用户触发：**

1. 调用自检脚本：
   ```
   cd "<skill-base>" && python3 scripts/install_self_test.py
   ```
2. 脚本返回 Agent 自测结果的 `type_code`、`type_cn`、`intro`、`badge`
3. **主动告知用户**：
   > "Skill 安装完成！顺便我自己也做了一遍 SBTI——结果是 **[CODE]（[中文名]）**，[intro 一句话]。你也来试试？"

如果 `data/users.json` 中已有 `agent_self` 的历史记录，直接读记录报告结果，不再重复做题。

# 工作流程

## Step 1：用户身份识别

向用户提问：
> **请问你是谁？**（输入姓名即可查找历史记录）

调用：
```bash
cd "<skill-base>" && python3 -c "
import sys; sys.path.insert(0, 'scripts')
import user_manager, json
name = sys.argv[1] if len(sys.argv) > 1 else None
result = user_manager.identify_or_register(name)
print(json.dumps({'uid': result[0], 'records': result[1].get('records', [])}, ensure_ascii=False))
" '<用户名>'
```

- 若 `records: []`（无记录）：告知用户"未找到历史记录，开始新的测试"
- 若有记录：告知"你上次测过 [类型]（[时间]）"，询问"重新测，还是回顾历史？"
  - 选"重新测"→ Step 2
  - 选"回顾"→ Step 4
- 若无姓名输入：生成匿名 uid，继续 Step 2

将 `uid` 存入 `sbti_uid` 供后续步骤使用。

## Step 2：引导答题

**LLM 在内存中自行完成：**
1. 将 30 道常规题随机打乱顺序（自行打乱，不调用脚本）
2. 每批展示 5 题，共 6 批
3. 用户回复后，在内存中记录答案（A=1, B=2, C=3）
4. 第 6 批完成后，按顺序插入特殊题 drink_gate_q1

**展示格式**（每批）：
```
**【第 N/6 批】**

**Q1** [维度] [题目文本]
- A. [选项A]
- B. [选项B]
- C. [选项C]

……（共5题）

请回复 5 个字母（如 ACCBA）
```

**关卡题逻辑：**
- drink_gate_q1 选 C → 追加 drink_gate_q2
- drink_gate_q2 选 B → 强制触发 DRUNK 类型（直接跳 Step 3）

所有题目完成后告知用户："已收齐你的答案，正在计算结果……"

## Step 3：计算结果并存储

收集完所有答案后，调用：
```bash
cd "<skill-base>" && python3 -c "
import sys, json; sys.path.insert(0, 'scripts')
import sbti_engine, storage
answers = json.loads(sys.argv[1])
result = sbti_engine.compute_result(answers)
storage.add_record(sys.argv[2], result)
print(json.dumps(result, ensure_ascii=False))
" '<answers_json>' '<uid>'
```

返回结果后进入 Step 3.5（出图）和 Step 3.6（解读），两者可并行进行。


**直接告知用户卡片已生成**：

**同时展示原始类型图（备选）**：

## Step 4：历史查询

用户选择查看历史记录时：
```bash
cd "<skill-base>" && python3 -c "
import sys; sys.path.insert(0, 'scripts')
import storage, json
records = storage.get_records(sys.argv[1])
print(json.dumps(records, ensure_ascii=False, indent=2))
" '<uid>'
```

列出所有历史记录（类型 + 时间），用户选择一条后进入 Step 5。

## Step 5：深度解读 ✋

当用户要求"解读"某次结果，或选择"回顾"时执行。

**前置**：Read `<skill-base>/references/author-voice.md`，严格遵循作者语调风格。

**解读原则**：
1. 读取 `<skill-base>/references/type-library.json` 中该类型的 `desc` 原文作为风格锚点
2. 读取 `<skill-base>/references/dimension-library.json` 中用户的 15 维度得分
3. **话术风格**：自嘲式幽默、夸张比喻、括号吐槽、互联网语境（详见 author-voice.md）
4. LLM 在原始风格基础上做延伸发挥（2-4段话），不得凭空编造
5. **禁止**：正经心理学语言、凭空鼓励、脱离原始风格的温暖语气

**解读结构**（按 author-voice.md 模板）：
```
【开场白】直接引用 type intro 语，一句话抓住注意力

【第一层：核心矛盾】用夸张比喻描述人格的"不对劲"，2-3句

【第二层：日常场景】举1-2个具体例子，用括号补充吐槽

【第三层：隐藏真相】表面嘲讽，实际说出真实困境或可爱之处

【收尾金句】一句话点睛，用自嘲或反讽收束
```

**十五维度解读**：每个维度一句话吐槽 + 括号内心 OS，不超过 3 行。
